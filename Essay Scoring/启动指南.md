# 智能作文评分系统 - 启动与使用指南

## 🚀 快速启动

### 前置要求

1. **Java 17+** - 确保已安装并配置环境变量
   ```bash
   java -version  # 应该显示 Java 17 或更高版本
   ```

2. **Maven 3.6+** - 确保已安装并配置环境变量
   ```bash
   mvn -version  # 应该显示 Maven 3.6 或更高版本
   ```

3. **DeepSeek API Key**（可选，未配置时使用模拟模式）
   - 访问：https://platform.deepseek.com/
   - 注册账号并获取 API Key

### 步骤 1：配置 API Key（可选）

编辑 `src/main/resources/application.yml` 文件：

```yaml
spring:
  cloud:
    ai:
      deepseek:
        api-key: your-api-key-here  # 替换为您的真实 DeepSeek API Key
```

**或者使用环境变量**（推荐，更安全）：

**Windows:**
```cmd
set DEEPSEEK_API_KEY=your-api-key-here
```

**Linux/Mac:**
```bash
export DEEPSEEK_API_KEY=your-api-key-here
```

> **注意**：如果不配置 API Key，系统会自动使用 MockChatClient（模拟模式），可以正常测试功能，但评分结果是模拟数据。

### 步骤 2：编译项目

在项目根目录执行：

```bash
mvn clean compile
```

如果编译成功，会看到 `BUILD SUCCESS` 提示。

### 步骤 3：启动项目

**方式一：使用 Maven 命令启动**
```bash
mvn spring-boot:run
```

**方式二：打包后运行**
```bash
# 打包
mvn clean package

# 运行
java -jar target/EssayScoring-0.0.1-SNAPSHOT.jar
```

### 步骤 4：访问系统

启动成功后，在浏览器中访问：

- **学生端**：http://localhost:8080/student.html
- **教师端**：http://localhost:8080/teacher.html
- **H2 数据库控制台**：http://localhost:8080/h2-console
  - JDBC URL: `jdbc:h2:mem:essayscoring`
  - 用户名: `sa`
  - 密码: （留空）

## 📖 使用说明

### 学生端使用流程

#### 1. 提交作文

1. 打开学生端页面：http://localhost:8080/student.html
2. 填写作文信息：
   - **学生姓名**：输入您的姓名
   - **学生ID**：输入学号或唯一标识
   - **作文标题**：输入作文标题
   - **年级**：选择对应的年级
   - **文体类型**：选择作文类型（记叙文、议论文等）
   - **作文内容**：在文本框中输入或粘贴作文内容

3. 点击 **"提交并评分"** 按钮

#### 2. 查看评分结果

提交后，系统会自动进行 AI 评分，显示：

- **总分**：四个维度的综合得分（满分100分）
- **各维度得分**：
  - 内容（满分30分）
  - 结构（满分25分）
  - 语言（满分25分）
  - 创意（满分20分）
- **雷达图**：可视化展示各维度得分情况
- **改进建议**：包含问题描述、改进建议、优化示例等

#### 3. 查看历史作文

可以通过 API 查看历史提交的作文：
```
GET http://localhost:8080/api/student/essays?studentId=您的学号
```

---

### 教师端使用流程

#### 1. 查看所有作文

1. 打开教师端页面：http://localhost:8080/teacher.html
2. 默认显示 **"作文管理"** 标签页
3. 可以看到所有学生提交的作文列表，包括：
   - 学生信息
   - 作文标题
   - 年级和文体
   - 字数统计
   - 总分
   - 提交时间

#### 2. 管理评分规则

1. 点击 **"评分规则"** 标签页
2. 可以：
   - **新增规则**：点击"新增规则"按钮，填写规则信息
   - **编辑规则**：点击对应规则的"编辑"按钮
   - **启用/禁用规则**：点击"启用"或"禁用"按钮
   - **删除规则**：点击"删除"按钮

**评分规则配置说明**：
- **规则名称**：规则的名称
- **适用年级**：该规则适用的年级（留空表示全部）
- **适用文体**：该规则适用的文体（留空表示全部）
- **评分维度**：content（内容）、structure（结构）、language（语言）、creativity（创意）
- **权重**：0-1之间的小数，表示该维度在总分中的权重
- **评分标准**：详细的评分标准描述

#### 3. 查看统计分析

1. 点击 **"统计分析"** 标签页
2. 查看系统统计信息：
   - 总作文数
   - 学生总数
   - 平均分

```http
GET /api/student/essay/{id}/result
```

#### 3. 查询学生的所有作文
```http
GET /api/student/essays?studentId=S001
```

### 教师端 API

#### 1. 获取所有作文
```http
GET /api/teacher/essays
```

#### 2. 获取所有评分规则
```http
GET /api/teacher/rules
```

#### 3. 创建评分规则
```http
POST /api/teacher/rules
Content-Type: application/json

{
  "ruleName": "内容评分规则",
  "dimension": "content",
  "weight": 0.3,
  "criteria": "主题明确、内容充实、观点正确",
  "enabled": true
}
```

#### 4. 更新评分规则
```http
PUT /api/teacher/rules/{id}
Content-Type: application/json

{
  "ruleName": "内容评分规则（更新）",
  "dimension": "content",
  "weight": 0.35,
  "criteria": "更新后的评分标准",
  "enabled": true
}
```

#### 5. 删除评分规则
```http
DELETE /api/teacher/rules/{id}
```

#### 6. 启用/禁用评分规则
```http
PUT /api/teacher/rules/{id}/toggle
```

#### 7. 获取统计信息
```http
GET /api/teacher/statistics
```

## 🛠️ 常见问题

### 1. 启动失败：端口被占用

**错误信息**：`Port 8080 is already in use`

**解决方法**：
- 修改 `application.yml` 中的端口：
  ```yaml
  server:
    port: 8081  # 改为其他端口
  ```
- 或者关闭占用 8080 端口的程序

### 2. 编译错误：找不到类

**解决方法**：
```bash
# 清理并重新编译
mvn clean compile

# 如果还有问题，删除本地仓库缓存
rm -rf ~/.m2/repository/com/study/ai
mvn clean install
```

### 3. AI 调用失败

**可能原因**：
- API Key 未配置或配置错误
- 网络连接问题
- API Key 余额不足

**解决方法**：
- 检查 API Key 是否正确配置
- 检查网络连接
- 查看控制台日志获取详细错误信息
- 如果 API Key 未配置，系统会自动使用 Mock 模式

### 4. 数据库连接问题

**H2 数据库连接信息**：
- JDBC URL: `jdbc:h2:mem:essayscoring`
- 用户名: `sa`
- 密码: （留空）

**访问 H2 控制台**：
http://localhost:8080/h2-console
