# H2 控制台访问问题排查指南

## 当前配置状态

已完成的配置：
1. ✅ `WebSecurityCustomizer` - 完全忽略 H2 控制台
2. ✅ `JwtAuthenticationFilter` - 跳过 H2 控制台路径
3. ✅ `SecurityFilterChain` - 允许 H2 控制台访问
4. ✅ CSRF 已禁用
5. ✅ 框架选项已禁用

## 排查步骤

### 步骤 1：确认应用已重启

**重要**：修改配置后必须重启应用才能生效！

1. 完全停止应用（Ctrl+C）
2. 等待几秒确保进程完全退出
3. 重新启动：
   ```bash
   mvn spring-boot:run
   ```

### 步骤 2：检查启动日志

启动时应该看到类似这样的日志：
```
H2 console available at '/h2-console'. Database available at 'jdbc:h2:file:./data/essayscoring'
```

如果没有看到，说明 H2 控制台可能没有正确启用。

### 步骤 3：清除浏览器缓存

1. 按 `Ctrl + Shift + Delete` 清除浏览器缓存
2. 或者使用无痕模式访问：
   ```
   http://localhost:8080/h2-console
   ```

### 步骤 4：检查端口是否正确

确认应用运行在 8080 端口：
- 查看启动日志中的端口信息
- 或访问：http://localhost:8080/ （应该能看到首页）

### 步骤 5：尝试直接访问完整 URL

尝试以下 URL：
```
http://localhost:8080/h2-console/
http://127.0.0.1:8080/h2-console
```

### 步骤 6：检查浏览器控制台

1. 按 `F12` 打开开发者工具
2. 查看 Console 和 Network 标签
3. 查看是否有错误信息

### 步骤 7：测试其他页面

访问以下页面，确认应用正常运行：
- http://localhost:8080/ （首页）
- http://localhost:8080/student.html （学生端）
- http://localhost:8080/teacher.html （教师端）

如果这些页面也无法访问，说明是 Spring Security 配置问题。
