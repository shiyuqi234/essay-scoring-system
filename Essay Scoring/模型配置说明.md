# 如何修改调用的AI模型

## 📋 模型配置位置

模型配置在 `src/main/resources/application.yml` 文件中。

## 🔧 修改方法

### 方法一：直接修改配置文件（推荐）

编辑 `src/main/resources/application.yml` 文件，找到以下部分：

```yaml
spring:
  cloud:
    ai:
      dashscope:
        api-key: ${DASHSCOPE_API_KEY:your-api-key-here}
        chat:
          options:
            model: qwen-turbo  # 👈 在这里修改模型名称
            temperature: 0.3   # 👈 温度参数（0-1，越低越稳定）
            max-tokens: 2000   # 👈 最大输出token数
```

### 方法二：使用环境变量

也可以通过环境变量设置：

**Windows:**
```cmd
set DASHSCOPE_MODEL=qwen-plus
set DASHSCOPE_TEMPERATURE=0.3
set DASHSCOPE_MAX_TOKENS=2000
```

**Linux/Mac:**
```bash
export DASHSCOPE_MODEL=qwen-plus
export DASHSCOPE_TEMPERATURE=0.3
export DASHSCOPE_MAX_TOKENS=2000
```

然后在 `application.yml` 中使用：
```yaml
spring:
  cloud:
    ai:
      dashscope:
        chat:
          options:
            model: ${DASHSCOPE_MODEL:qwen-turbo}
            temperature: ${DASHSCOPE_TEMPERATURE:0.3}
            max-tokens: ${DASHSCOPE_MAX_TOKENS:2000}
```

## 🤖 可用的模型列表

### 通义千问（Qwen）系列

| 模型名称 | 说明 | 适用场景 |
|---------|------|---------|
| `qwen-turbo` | 快速响应版本 | 适合快速评分，成本较低 |
| `qwen-plus` | 增强版本 | 平衡性能和效果 |
| `qwen-max` | 最强版本 | 适合高质量评分，效果最好 |
| `qwen-max-longcontext` | 长文本版本 | 适合超长作文（支持更长上下文） |

### 通义千问最新系列（Qwen2.5/Qwen3）

| 模型名称 | 说明 |
|---------|------|
| `qwen2.5-72b-instruct` | Qwen2.5 72B 指令版本 |
| `qwen2.5-32b-instruct` | Qwen2.5 32B 指令版本 |
| `qwen2.5-14b-instruct` | Qwen2.5 14B 指令版本 |
| `qwen2.5-7b-instruct` | Qwen2.5 7B 指令版本 |
| `qwen3-max` | Qwen3 最强版本 |

> **注意**：模型名称可能会更新，请以阿里云百炼平台官方文档为准。

## ⚙️ 参数说明

### model（模型名称）
- **默认值**：`qwen-turbo`
- **说明**：要使用的AI模型名称
- **示例**：`qwen-plus`、`qwen-max`、`qwen3-max`

### temperature（温度参数）
- **默认值**：`0.3`
- **范围**：0.0 - 1.0
- **说明**：
  - 值越低（如 0.1-0.3）：输出更稳定、更确定，适合评分任务
  - 值越高（如 0.7-1.0）：输出更随机、更有创意
- **推荐值**：评分任务建议使用 `0.3` 或更低

### max-tokens（最大输出长度）
- **默认值**：`2000`
- **说明**：AI返回内容的最大token数
- **建议**：
  - 评分结果：2000 足够
  - 详细反馈：可以增加到 3000-4000

## 🔄 修改后如何生效

1. **修改配置文件**后，需要**重启应用**才能生效
2. 重启命令：
   ```bash
   # 停止当前运行的应用（Ctrl+C）
   # 然后重新启动
   mvn spring-boot:run
   ```

## 💡 模型选择建议

### 根据使用场景选择：

1. **开发/测试环境**
   - 推荐：`qwen-turbo`
   - 原因：响应快、成本低

2. **生产环境（追求质量）**
   - 推荐：`qwen-max` 或 `qwen3-max`
   - 原因：评分更准确、反馈更详细

3. **大批量评分（成本敏感）**
   - 推荐：`qwen-turbo` 或 `qwen-plus`
   - 原因：平衡成本和效果

4. **超长作文（>2000字）**
   - 推荐：`qwen-max-longcontext`
   - 原因：支持更长上下文

---

## ⚠️ 注意事项

1. **API Key权限**：确保您的API Key有权限使用所选模型
2. **成本差异**：不同模型的调用成本不同，`qwen-max` 比 `qwen-turbo` 更贵
3. **响应速度**：模型越大，响应可能越慢
4. **模型可用性**：某些模型可能在某些区域不可用，请查看阿里云文档

## 🔍 验证配置

修改后，可以通过以下方式验证：

1. **查看启动日志**：应用启动时会显示配置信息
2. **测试提交**：提交一篇作文，查看评分结果
3. **查看日志**：在日志中可以看到实际调用的模型
