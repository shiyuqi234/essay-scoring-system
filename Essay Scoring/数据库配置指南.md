# 数据库配置指南

## 📊 当前配置说明

项目当前使用的是 **H2 内存数据库**，这意味着：

✅ **优点**：
- 不需要手动创建数据库
- 应用启动时自动创建
- 零配置，开箱即用
- 适合开发和测试

⚠️ **缺点**：
- 数据存储在内存中
- 应用重启后数据会丢失
- 不适合生产环境

---

## 🎯 方案选择

### 方案一：H2 文件数据库（推荐用于开发）

**特点**：
- ✅ 数据持久化到文件
- ✅ 不需要手动创建数据库
- ✅ 应用启动时自动创建
- ✅ 适合开发和测试环境

**配置步骤**：

1. 修改 `src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    # 改为文件数据库（数据会保存到项目目录下的 essayscoring.mv.db 文件）
    url: jdbc:h2:file:./data/essayscoring
    driver-class-name: org.h2.Driver
    username: sa
    password: 
  
  jpa:
    hibernate:
      ddl-auto: update  # 自动创建表结构
    show-sql: true
    properties:
      hibernate:
        dialect: org.hibernate.dialect.H2Dialect
        format_sql: true
  
  h2:
    console:
      enabled: true
      path: /h2-console
```

2. 创建数据目录（可选，应用会自动创建）：
```bash
mkdir data
```

3. 重启应用，数据库文件会自动创建在 `data/essayscoring.mv.db`

---

### 方案二：MySQL 数据库（推荐用于生产）

**特点**：
- ✅ 数据持久化
- ✅ 适合生产环境
- ✅ 支持多用户并发
- ⚠️ 需要手动创建数据库

**配置步骤**：

#### 步骤 1：安装 MySQL

**Windows**：
1. 下载 MySQL：https://dev.mysql.com/downloads/mysql/
2. 安装 MySQL Server
3. 记住 root 密码

**Linux (Ubuntu/Debian)**：
```bash
sudo apt update
sudo apt install mysql-server
sudo mysql_secure_installation
```

**Mac**：
```bash
brew install mysql
brew services start mysql
```

#### 步骤 2：创建数据库

**方式一：使用命令行**

1. 登录 MySQL：
```bash
mysql -u root -p
```

2. 创建数据库：
```sql
CREATE DATABASE essayscoring CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

3. 创建用户（可选，推荐）：
```sql
CREATE USER 'essay_user'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON essayscoring.* TO 'essay_user'@'localhost';
FLUSH PRIVILEGES;
```

4. 退出：
```sql
EXIT;
```

**方式二：使用图形化工具**

1. 打开 MySQL Workbench、Navicat 或 phpMyAdmin
2. 连接到 MySQL 服务器
3. 创建新数据库：
   - 数据库名：`essayscoring`
   - 字符集：`utf8mb4`
   - 排序规则：`utf8mb4_unicode_ci`

#### 步骤 3：修改项目配置

编辑 `src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/essayscoring?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    driver-class-name: com.mysql.cj.jdbc.Driver
    username: root  # 或使用上面创建的用户名 essay_user
    password: your_password  # 替换为您的 MySQL 密码
  
  jpa:
    hibernate:
      ddl-auto: update  # 自动创建表结构
    show-sql: true
    properties:
      hibernate:
        dialect: org.hibernate.dialect.MySQL8Dialect
        format_sql: true

# 注释掉 H2 控制台配置（如果使用 MySQL）
# h2:
#   console:
#     enabled: false
```

#### 步骤 4：验证配置

1. 确保 `pom.xml` 中已包含 MySQL 驱动（已包含）：
```xml
<dependency>
    <groupId>com.mysql</groupId>
    <artifactId>mysql-connector-j</artifactId>
    <scope>runtime</scope>
</dependency>
```

2. 启动应用：
```bash
mvn spring-boot:run
```

3. 如果启动成功，表结构会自动创建。可以通过 MySQL 客户端查看：
```sql
USE essayscoring;
SHOW TABLES;
```

应该能看到以下表：
- `essays` - 作文表
- `scores` - 评分表
- `feedbacks` - 反馈表
- `scoring_rules` - 评分规则表

## 🔍 数据库表结构说明

系统会自动创建以下表：

### 1. essays（作文表）
- `id` - 主键
- `student_name` - 学生姓名
- `student_id` - 学生ID
- `title` - 作文标题
- `content` - 作文内容
- `grade` - 年级
- `essay_type` - 文体类型
- `word_count` - 字数
- `submit_time` - 提交时间
- `create_time` - 创建时间
- `update_time` - 更新时间

### 2. scores（评分表）
- `id` - 主键
- `essay_id` - 作文ID（外键）
- `total_score` - 总分
- `content_score` - 内容得分
- `structure_score` - 结构得分
- `language_score` - 语言得分
- `creativity_score` - 创意得分
- `content_comment` - 内容评价
- `structure_comment` - 结构评价
- `language_comment` - 语言评价
- `creativity_comment` - 创意评价
- `overall_comment` - 综合评语
- `score_time` - 评分时间
- `model_version` - 模型版本

### 3. feedbacks（反馈表）
- `id` - 主键
- `essay_id` - 作文ID（外键）
- `feedback_type` - 反馈类型
- `position` - 问题位置
- `issue` - 问题描述
- `suggestion` - 改进建议
- `improved_example` - 优化示例
- `severity` - 严重程度
- `create_time` - 创建时间

### 4. scoring_rules（评分规则表）
- `id` - 主键
- `rule_name` - 规则名称
- `applicable_grade` - 适用年级
- `applicable_essay_type` - 适用文体
- `dimension` - 评分维度
- `weight` - 权重
- `criteria` - 评分标准
- `enabled` - 是否启用
- `create_time` - 创建时间
- `update_time` - 更新时间
- `created_by` - 创建人

## 🛠️ 数据库管理工具

### H2 数据库控制台

如果使用 H2 数据库，可以通过 Web 控制台管理：

1. 启动应用
2. 访问：http://localhost:8080/h2-console
3. 连接信息：
   - JDBC URL: `jdbc:h2:mem:essayscoring`（内存数据库）
   - 或 `jdbc:h2:file:./data/essayscoring`（文件数据库）
   - 用户名: `sa`
   - 密码: （留空）

推荐使用以下工具管理 MySQL：

1. **MySQL Workbench**（官方工具）
   - 下载：https://dev.mysql.com/downloads/workbench/

   - 功能强大，界面友好

3. **phpMyAdmin**（Web 界面）
   - 适合 Web 服务器环境

4. **DBeaver**（免费开源）
   - 支持多种数据库
   - 下载：https://dbeaver.io/

## 📝 配置示例

### 开发环境（H2 文件数据库）

```yaml
spring:
  datasource:
    url: jdbc:h2:file:./data/essayscoring
    driver-class-name: org.h2.Driver
    username: sa
    password: 
  jpa:
    hibernate:
      ddl-auto: update
```

### 生产环境（MySQL）

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/essayscoring?useUnicode=true&characterEncoding=utf8&useSSL=true&serverTimezone=Asia/Shanghai
    driver-class-name: com.mysql.cj.jdbc.Driver
    username: essay_user
    password: ${DB_PASSWORD:your_secure_password}
  jpa:
    hibernate:
      ddl-auto: validate  # 生产环境建议使用 validate
```

---

## ⚠️ 注意事项

1. **生产环境安全**：
   - 不要在配置文件中硬编码密码
   - 使用环境变量或配置中心管理敏感信息
   - 示例：`password: ${DB_PASSWORD}`

2. **数据备份**：
   - 定期备份数据库
   - MySQL 备份命令：
     ```bash
     mysqldump -u root -p essayscoring > backup.sql
     ```

3. **字符集设置**：
   - 确保使用 `utf8mb4` 字符集以支持中文和 emoji

4. **连接池配置**（可选）：
   ```yaml
   spring:
     datasource:
       hikari:
         maximum-pool-size: 10
         minimum-idle: 5
         connection-timeout: 30000
   ```

## ❓ 常见问题

### Q: 应用启动时提示数据库连接失败？

A: 检查：
1. MySQL 服务是否启动
2. 数据库名称是否正确
3. 用户名和密码是否正确
4. 端口号是否正确（默认 3306）

### Q: 表结构没有自动创建？

A: 确保配置了：
```yaml
jpa:
  hibernate:
    ddl-auto: update
```

### Q: 如何查看数据库中的数据？

A: 
- H2：访问 http://localhost:8080/h2-console
- MySQL：使用 MySQL Workbench 或其他工具连接

### Q: 如何重置数据库？

A: 
- H2 文件数据库：删除 `data/essayscoring.mv.db` 文件
- MySQL：删除并重新创建数据库
