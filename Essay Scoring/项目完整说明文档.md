## 📋 项目概述

### 项目简介

智能作文评分与反馈系统是一个基于 Spring Boot 和阿里云百炼平台（DashScope）的 Web 应用系统，融合 NLP 与 AI 大模型能力，能够自动对学生的作文进行多维度智能评分，并提供结构化的反馈和改进建议。

### 核心价值

- ✅ **AI 智能评分**：基于通义千问大模型，实现多维度自动评分
- ✅ **结构化反馈**：提供语法检查、表达优化、结构改进等详细反馈
- ✅ **角色权限管理**：支持学生和教师两种角色，权限分离
- ✅ **历史记录查询**：学生可查看历史作文及评分记录
- ✅ **数据可视化**：雷达图展示各维度得分情况
- ✅ **评分规则管理**：教师可自定义评分规则和权重

## 🚀 快速开始

### 环境要求

- **JDK**: 17 或更高版本
- **Maven**: 3.6 或更高版本
- **MySQL**: 8.0 或更高版本（生产环境）
- **H2 Database**: 可选（开发环境）
- **阿里云百炼平台 API Key**: 用于 AI 评分（可选，未配置时使用模拟模式）

### 安装步骤

#### 1. 克隆项目

```bash
git clone <repository-url>
cd "Essay Scoring"
```

#### 2. 配置数据库

**方式一：使用 MySQL（推荐生产环境）**

1. 创建数据库：
```sql
CREATE DATABASE essayscoring CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

2. 修改 `src/main/resources/application.yml`：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/essayscoring?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    driver-class-name: com.mysql.cj.jdbc.Driver
    username: root
    password: your_password
```

**方式二：使用 H2（开发环境）**

修改 `src/main/resources/application.yml`：
```yaml
spring:
  datasource:
    url: jdbc:h2:file:./data/essayscoring
    driver-class-name: org.h2.Driver
    username: sa
    password: 
```

#### 3. 配置 AI 模型 API Key

编辑 `src/main/resources/application.yml`：

```yaml
spring:
  cloud:
    ai:
      dashscope:
        api-key: ${DASHSCOPE_API_KEY:your-api-key-here}
        chat:
          options:
            model: qwen3-max  # 可选：qwen-turbo, qwen-plus, qwen-max, qwen3-max
            temperature: 0.3
            max-tokens: 2000
```

**获取 API Key**：
1. 访问 [阿里云百炼平台](https://dashscope.aliyun.com/)
2. 注册账号并登录
3. 在控制台获取 API Key

**或使用环境变量**（推荐）：
```bash
# Windows
set DASHSCOPE_API_KEY=your-api-key-here

# Linux/Mac
export DASHSCOPE_API_KEY=your-api-key-here
```

#### 4. 配置 JWT 密钥

编辑 `src/main/resources/application.yml`：

```yaml
jwt:
  secret: your-base64-encoded-secret-key  # 建议使用更长的随机字符串
  expiration-ms: 86400000  # 24 小时
```

#### 5. 编译项目

```bash
mvn clean compile
```

#### 6. 启动项目

```bash
mvn spring-boot:run
```

#### 7. 访问系统

- **学生端**: http://localhost:8080/student.html
- **教师端**: http://localhost:8080/teacher.html
- **首页**: http://localhost:8080/index.html

## ⚙️ 配置说明

### application.yml 完整配置示例

```yaml
server:
  port: 8080

spring:
  application:
    name: essay-scoring-system
  
  # 数据源配置
  datasource:
    url: jdbc:mysql://localhost:3306/essayscoring?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    driver-class-name: com.mysql.cj.jdbc.Driver
    username: root
    password: your_password
  
  # JPA 配置
  jpa:
    hibernate:
      ddl-auto: update  # 自动创建/更新表结构
    show-sql: true  # 显示 SQL 语句
    properties:
      hibernate:
        dialect: org.hibernate.dialect.MySQL8Dialect
        format_sql: true
  
  # 阿里云百炼平台配置
  cloud:
    ai:
      dashscope:
        api-key: ${DASHSCOPE_API_KEY:your-api-key-here}
        chat:
          options:
            model: qwen3-max
            temperature: 0.3
            max-tokens: 2000

# JWT 配置
jwt:
  secret: your-base64-encoded-secret-key
  expiration-ms: 86400000  # 24 小时

# 日志配置
logging:
  level:
    root: INFO
    com.study.ai.essayscoring: DEBUG
```

### 配置项说明

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `server.port` | 服务端口 | 8080 |
| `spring.datasource.url` | 数据库连接 URL | - |
| `spring.datasource.username` | 数据库用户名 | root |
| `spring.datasource.password` | 数据库密码 | - |
| `spring.jpa.hibernate.ddl-auto` | 表结构自动更新策略 | update |
| `spring.cloud.ai.dashscope.api-key` | DashScope API Key | - |
| `spring.cloud.ai.dashscope.chat.options.model` | AI 模型名称 | qwen3-max |
| `spring.cloud.ai.dashscope.chat.options.temperature` | 温度参数（0-1） | 0.3 |
| `spring.cloud.ai.dashscope.chat.options.max-tokens` | 最大输出 token 数 | 2000 |
| `jwt.secret` | JWT 密钥（Base64 编码） | - |
| `jwt.expiration-ms` | Token 过期时间（毫秒） | 86400000 |

### 表关系图

```

  (独立表，用于管理评分规则)
```

## 🔐 用户认证与授权

### 认证流程

1. **用户注册**
   - 用户填写注册信息（用户名、密码、角色等）
   - 后端验证用户名唯一性
   - 密码使用 BCrypt 加密存储
   - 创建用户账户

2. **用户登录**
   - 用户提交用户名和密码
   - Spring Security 验证用户凭证
   - 验证成功后生成 JWT Token
   - 返回 Token 给前端

3. **Token 使用**
   - 前端将 Token 存储在 localStorage
   - 每次请求在 `Authorization` 头中携带 Token
   - `JwtAuthenticationFilter` 拦截请求并验证 Token
   - 验证成功后设置 `SecurityContext`

### 授权规则

| 路径 | 权限要求 |
|------|----------|
| `/api/auth/**` | 允许所有用户（未认证也可访问） |
| `/api/student/**` | 需要认证（学生或教师） |
| `/api/teacher/**` | 需要教师角色 |
| `/`, `/index.html`, `/student.html`, `/teacher.html` | 允许所有用户 |
| `/css/**`, `/js/**`, `/images/**` | 允许所有用户 |

### JWT Token 结构

```json
{
  "sub": "username",
  "role": "STUDENT",
  "iat": 1704672000,
  "exp": 1704758400
}
```

- `sub`: 用户名（Subject）
- `role`: 用户角色
- `iat`: 签发时间（Issued At）
- `exp`: 过期时间（Expiration）

### 安全特性

- ✅ 密码使用 BCrypt 加密存储
- ✅ JWT Token 签名验证
- ✅ Token 过期时间控制（默认 24 小时）
- ✅ 角色权限分离（学生/教师）
- ✅ CORS 跨域支持
- ✅ CSRF 保护（已禁用，因为使用 JWT）

## 🚀 部署指南

### 开发环境部署

1. **使用 H2 文件数据库**
   - 修改 `application.yml` 使用 H2
   - 启动应用，数据库文件自动创建

2. **使用 MySQL**
   - 创建数据库
   - 配置连接信息
   - 启动应用，表结构自动创建

### 生产环境部署

#### 1. 打包应用

```bash
mvn clean package
```

生成文件：`target/EssayScoring-0.0.1-SNAPSHOT.jar`

#### 2. 配置生产环境

创建 `application-prod.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://your-db-host:3306/essayscoring?useUnicode=true&characterEncoding=utf8&useSSL=true&serverTimezone=Asia/Shanghai
    username: ${DB_USERNAME}
    password: ${DB_PASSWORD}
  jpa:
    hibernate:
      ddl-auto: validate  # 生产环境使用 validate

jwt:
  secret: ${JWT_SECRET}  # 使用环境变量

logging:
  level:
    root: INFO
    com.study.ai.essayscoring: INFO
```

#### 3. 运行应用

```bash
java -jar -Dspring.profiles.active=prod EssayScoring-0.0.1-SNAPSHOT.jar
```

#### 4. 使用 systemd 管理（Linux）

创建 `/etc/systemd/system/essay-scoring.service`：

```ini
[Unit]
Description=Essay Scoring Application
After=syslog.target

[Service]
User=your-user
ExecStart=/usr/bin/java -jar /path/to/EssayScoring-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod
SuccessExitStatus=143

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl start essay-scoring
sudo systemctl enable essay-scoring
```

#### 5. 使用 Nginx 反向代理（可选）

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### Docker 部署（可选）

创建 `Dockerfile`：

```dockerfile
FROM openjdk:17-jdk-slim
WORKDIR /app
COPY target/EssayScoring-0.0.1-SNAPSHOT.jar app.jar

```

## ❓ 常见问题

### 1. 启动失败：端口被占用

**错误**: `Port 8080 is already in use`

**解决**: 
- 修改 `application.yml` 中的端口号
- 或关闭占用 8080 端口的程序

### 2. 数据库连接失败

**错误**: `Communications link failure`

**解决**:
- 检查 MySQL 服务是否启动
- 检查数据库名称、用户名、密码是否正确
- 检查防火墙设置

### 3. AI 调用失败

**错误**: `API call failed`

**解决**:
- 检查 API Key 是否正确配置
- 检查网络连接
- 检查 API Key 余额
- 查看日志获取详细错误信息

### 4. JWT Token 无效

**错误**: `JWT signature does not match`

**解决**:
- 检查 JWT 密钥配置是否正确
- 确认 Token 未过期
- 重新登录获取新 Token

### 5. 权限不足

**错误**: `Access Denied`

**解决**:
- 确认用户角色正确（学生/教师）
- 确认 Token 中包含正确的角色信息
- 检查 `SecurityConfig` 中的权限配置

### 6. 前端无法访问后端 API

**错误**: `CORS error`

**解决**:
- 检查 `SecurityConfig` 中 CORS 配置
- 确认前端页面和后端在同一域名或已配置跨域

### 7. 数据库表未自动创建

**解决**:
- 确认 `spring.jpa.hibernate.ddl-auto=update`
- 检查数据库用户是否有 CREATE TABLE 权限
- 查看启动日志中的错误信息
